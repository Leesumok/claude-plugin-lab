---

인자: $ARGUMENTS

위 인자를 파싱해줘. 형식은 다음 중 하나:
- (빈 값) — 브랜치명, 커밋 메시지 모두 없음 (현재 브랜치에서 커밋)
- `<브랜치명>` — 브랜치명만 제공 (커밋 메시지는 자동 생성)
- `<브랜치명> <커밋메시지>` — 둘 다 제공

다음을 순서대로 수행해줘:

1. `git status`로 현재 상태를 확인해줘.

2. 대상 브랜치를 결정해줘:
   - 브랜치명이 제공된 경우: 해당 브랜치명을 대상으로 사용해줘.
   - 브랜치명이 없는 경우: `git branch --show-current`로 현재 브랜치를 대상으로 사용해줘.

3. 대상 브랜치가 `main` 또는 `develop`이면 사용자에게 확인해줘:
   > 대상 브랜치가 `<브랜치명>`입니다. 이 브랜치에 직접 커밋/푸시할까요, 아니면 새 브랜치명을 입력하시겠습니까?
   - 사용자가 새 브랜치명을 입력하면 해당 브랜치를 대상으로 변경해줘.
   - 사용자가 현재 브랜치에 커밋하겠다고 하면 그대로 진행해줘.

4. 대상 브랜치로 전환해줘:
   - `git branch --list <브랜치명>`으로 존재 여부를 확인해줘.
   - 이미 존재하면 `git checkout <브랜치명>`
   - 없으면 `git checkout -b <브랜치명>`

5. 변경사항을 커밋해줘:
   - `git add`로 관련 파일 스테이징 (민감한 파일 제외)
   - 커밋 메시지가 없으면 변경사항을 분석해서 conventional commit 형식으로 자동 생성
   - `git commit`

6. 리모트에 푸시해줘:
   - `git push -u origin <브랜치명>`

7. 결과를 요약해줘 (브랜치명, 커밋 해시, 푸시 상태).

8. 사용자에게 PR을 생성할지 물어봐줘. 사용자가 동의하면 `/pull-request` 스킬을 실행해줘.
