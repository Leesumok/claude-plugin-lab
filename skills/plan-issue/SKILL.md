---

작업 주제: $ARGUMENTS

위 주제에 대해 다음을 수행해줘:

1. 작업계획을 먼저 수립해줘:
   - 목표 정의
   - 필요한 변경사항 목록 (파일 단위)
   - 예상 작업 단계 (체크리스트)
   - 주의사항 및 의존성

2. 작업에 적합한 브랜치 이름을 정해줘:
   - 형식: `feat/`, `fix/`, `refactor/` 등 conventional 접두사 + 간결한 설명
   - 예: `feat/login-api`, `fix/null-pointer-exception`

3. 작업계획이 완성되면 GitHub 이슈를 생성해줘:
   - `gh issue create` 사용
   - 제목: 작업 주제를 간결하게 요약
   - 본문: 위에서 수립한 작업계획을 마크다운으로 작성하되, 본문 하단에 `## 브랜치` 섹션을 추가하고 추천 브랜치 이름을 명시
   - 라벨이 있으면 적절히 추가

4. 생성된 이슈 번호, URL, 추천 브랜치 이름을 알려줘.
