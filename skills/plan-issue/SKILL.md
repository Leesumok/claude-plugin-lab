---

작업 주제: $ARGUMENTS

위 주제에 대해 다음을 수행해줘:

1. 작업계획을 먼저 수립해줘:
   - 목표 정의
   - 필요한 변경사항 목록 (파일 단위)
   - 예상 작업 단계 (체크리스트)
   - 주의사항 및 의존성

2. 작업계획이 완성되면 GitHub 이슈를 생성해줘:
   - `gh issue create` 사용
   - 제목: 작업 주제를 간결하게 요약
   - 본문: 위에서 수립한 작업계획을 마크다운으로 작성
   - 라벨이 있으면 적절히 추가

3. 생성된 이슈 번호와 URL을 알려줘.
