args_description: "<이슈번호>"
---

입력: $ARGUMENTS

위 인자에서 이슈 번호를 파싱해줘 (필수).

다음을 순서대로 수행해줘:

1. `gh issue view <이슈번호>` 로 이슈 내용을 확인해줘.

2. feature 브랜치를 결정해줘:
   - 이슈 본문에서 `## 브랜치` 섹션을 찾아줘.
   - 브랜치 이름이 하나면 그대로 사용.
   - 브랜치 이름이 여러 개면 작업 내용에 가장 적합한 것을 골라서 사용자에게 확인받아줘.
   - `## 브랜치` 섹션이 없거나 브랜치 이름을 찾을 수 없으면, 사용자에게 브랜치 이름을 직접 입력받아줘.

3. `git checkout -b <브랜치이름>` 으로 브랜치 생성 및 전환해줘.

4. 이슈에 작성된 작업계획대로 코드를 작성해줘:
   - 기존 코드 패턴과 컨벤션을 따를 것
   - 필요한 테스트가 있으면 함께 작성
   - 변경사항을 설명하면서 진행

5. 완료 후 생성된 브랜치 이름과 변경된 파일 목록을 요약해줘.

6. 사용자에게 커밋 및 푸시를 진행할지 물어봐줘. 사용자가 동의하면 `/branch-commit` 스킬을 실행해줘.
