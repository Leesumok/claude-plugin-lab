name: code-gen
description: 이슈 번호 기반 feature 브랜치 생성 + 코드 작성
---

인자 파싱:
- 첫 번째 인자(필수): 이슈 번호
- 선택 옵션:
  - `--jira <티켓번호>`: Jira 티켓 번호로 브랜치 생성 (예: `--jira PROJ-123` → `feat/PROJ-123-이슈설명`)
  - `--branch-rule <규칙>`: 브랜치 네이밍 규칙 직접 지정 (예: `--branch-rule "feature/{jira}-{desc}"`)
- 옵션이 없으면 이슈 본문의 `## 브랜치` 섹션에서 브랜치 이름을 가져옴. 없을 경우 이슈 제목 기반으로 자동 생성

입력: $ARGUMENTS

위 인자에 대해 다음을 수행해줘:

1. `gh issue view <이슈번호>` 로 이슈 내용을 확인해줘.

2. feature 브랜치를 생성해줘:
   - `--jira` 옵션이 있으면: `feat/<JIRA티켓번호>-<이슈요약>` 형식으로 브랜치 생성
   - `--branch-rule` 옵션이 있으면: 해당 규칙에 따라 브랜치 이름 생성
   - 옵션이 없으면: 이슈 본문의 `## 브랜치` 섹션에 명시된 이름 사용, 없으면 `feat/<이슈번호>-<이슈요약>` 형식
   - `git checkout -b <브랜치이름>` 으로 브랜치 생성 및 전환

3. 이슈에 작성된 작업계획을 분석해줘:
   - 어떤 파일을 생성/수정해야 하는지
   - 각 파일에서 어떤 변경이 필요한지

4. 작업계획에 따라 코드를 작성해줘:
   - 기존 코드 패턴과 컨벤션을 따를 것
   - 필요한 테스트가 있으면 함께 작성
   - 변경사항을 설명하면서 진행

5. 완료 후 생성된 브랜치 이름과 변경된 파일 목록을 요약해줘.
